Laboratory Orders

üÜï New in Version 1.1:

Lab orders now support CPT code multiselect (80+ common tests), recurring orders, future scheduling, and enhanced print functionality!

AureonCare's laboratory order module streamlines the process of ordering lab tests, tracking results, and managing laboratory workflows. The system integrates with major lab networks for electronic order transmission and result delivery.

Accessing Lab Orders

From Patient Chart

  1. Open patient's chart
  2. Go to "Laboratory" tab
  3. Click "+ New Lab Order" button

From Clinical Encounter

  1. During patient visit or while documenting a note
  2. Click "Order Labs" in the encounter toolbar
  3. Patient context is automatically populated

Creating a Lab Order

Step 1: Basic Information

Field Description Required
Patient Patient receiving the lab tests (pre-populated from context) Yes
Ordering Provider Provider ordering the tests (usually current user) Yes
Order Date Date order is placed (defaults to today) Yes
Priority Routine, STAT, or ASAP Yes
Clinical Indication Reason for ordering tests (ICD-10 diagnosis) Yes

Step 2: Select Lab Tests (CPT Codes)

New Multi-Select Interface: You can now select multiple lab tests in one order!

Using CPT Code Multiselect

  1. Click in the "Select Tests" field
  2. A dropdown appears showing 80+ common lab tests organized by category
  3. Browse by category or start typing to search:
    • Test name (e.g., "CBC", "Glucose")
    • CPT code (e.g., "85025", "82947")
    • Panel name (e.g., "Basic Metabolic Panel")
  4. Click to select multiple tests - they appear as chips/tags
  5. Click the X on any chip to remove a test
  6. Selected tests show with their CPT codes

Recurring Orders (New!)

For chronic condition monitoring, you can now schedule recurring lab orders:

  1. Toggle "Recurring Order" switch
  2. Select frequency:
    • Daily
    • Weekly (specify day of week)
    • Bi-weekly
    • Monthly (specify day of month)
    • Quarterly
    • Custom interval
  3. Set start date and end date (or number of occurrences)
  4. System automatically generates future orders on the schedule
üìÖ Example Use Cases for Recurring Orders:
  • Monthly INR for warfarin patients
  • Quarterly A1C for diabetic patients
  • Weekly drug levels for patients on certain medications

Step 4: Laboratory Selection

  1. Preferred Lab: Patient's preferred laboratory (if on file)
  2. Search Labs: Find by name, location, or specialty
  3. In-Network Labs: Filter to show only in-network labs for patient's insurance
  4. Lab Capabilities: Ensure lab performs the ordered tests

Step 5: Results Distribution

Result Recipients (New Multi-Select):

  • Select multiple recipients to receive results automatically
  • Options:
    • Ordering Provider: You (auto-selected)
    • Primary Care Physician: Patient's PCP
    • Specialist: Select from patient's care team
    • Patient Portal: Post to patient's portal when ready
    • Other Provider: Search and add any provider
üí° Patient Portal Integration:

When "Patient Portal" is selected, results are automatically posted to the patient's portal with provider notes explaining the findings.

Step 6: Review and Submit

  1. Review all order details
  2. Verify selected tests and CPT codes
  3. Confirm laboratory and collection method
  4. Check result recipients
  5. Click "Submit Order"

After Submission

  • Order is transmitted electronically to laboratory (if supported)
  • Requisition form is generated and can be printed
  • Patient receives instructions via portal/email/SMS
  • Order appears in pending labs list

Printing Lab Orders

Enhanced Print Functionality:

Printing Options

  1. After creating order, click "Print" button
  2. Choose print format:
    • Full Requisition: Complete lab requisition form with all details
    • Patient Instructions: Patient-friendly prep instructions
    • Both: Requisition + instructions on separate pages
  3. Preview before printing
  4. Print directly or save as PDF

Requisition Form Includes

  • Patient demographics and insurance
  • All ordered tests with CPT codes
  • ICD-10 diagnosis codes
  • Provider information and signature
  • Collection instructions
  • Laboratory barcode (if applicable)

Managing Lab Orders

Order Status

Status Meaning

Scheduled Collection appointment scheduled

Cancelled Order cancelled before collection

Cancelling an Order

  1. Go to Patient Chart > Laboratory
  2. Find the pending order
  3. Click "Cancel Order"
  4. Select reason and add notes
  5. Confirm cancellation

Note: Orders can only be cancelled before specimen collection.

Modifying an Order

To change a pending order:

  • Before Submission to Lab: Click "Edit" and update details
  • After Submission: Cancel original order and create a new one

Viewing Lab Results

When Results Arrive

  1. System notification alerts you to new results
  2. Go to Patient Chart > Laboratory
  3. Click on the completed order
  4. Review results panel showing:
    • Test name and value
    • Reference range
    • Abnormal flag (H, L, Critical)
    • Result date/time
    • Performing lab

Abnormal Results

Abnormal results are highlighted:

  • üî¥ Critical (H/L): Red text - requires immediate attention
  • üü° Abnormal: Yellow background - outside reference range
  • ‚ö™ Normal: Standard display - within reference range
‚ö†Ô∏è Critical Values:

Critical lab values trigger immediate alerts to ordering provider via SMS, email, and system notification. You must acknowledge receipt and document follow-up action.

Documenting Result Review

  1. After reviewing results, click "Sign Results"
  2. Add interpretation notes
  3. Document follow-up plan:
    • No action needed
    • Retest scheduled
    • Treatment adjustment
    • Specialist referral
    • Patient notified
  4. Apply electronic signature

Result Notifications to Patients

Automatic Portal Posting

If "Patient Portal" was selected as recipient:

  • Results post to portal automatically when reviewed and signed
  • Patient receives notification email/SMS
  • Provider notes and interpretation included
  • Critical results withheld from portal until provider contacts patient

Manual Patient Notification

  1. Review results in patient chart
  2. Click "Notify Patient"
  3. Choose method:
    • Portal message with results
    • Secure email summary
    • Phone call (log in chart)
    • Schedule follow-up appointment
  4. Document notification in patient record

Lab Reports and Analytics

Access lab analytics from Reports > Laboratory:

  • Pending Orders Report: All orders awaiting collection or results
  • Unsigned Results: Results requiring provider review
  • Critical Values Log: History of all critical results
  • Turnaround Time: Average time from order to result by lab
  • Most Ordered Tests: Frequency of specific lab tests
  • Cost Analysis: Lab ordering costs by provider or department

Best Practices

üí° Lab Ordering Best Practices:
  • Always Include Diagnosis: Link orders to clinical indication for insurance approval
  • Use Panels Wisely: Order panels when appropriate to reduce costs
  • Check for Duplicates: Review recent labs before ordering to avoid unnecessary repeats
  • Set Correct Priority: Reserve STAT for true emergencies
  • Verify Fasting Requirements: Ensure patient is properly prepared
  • Use Recurring Orders: For chronic disease monitoring
  • Document Review: Sign all results promptly
  • Follow Up Abnormals: Always document action on abnormal results
  • Notify Patients: Communicate results in timely manner

Troubleshooting

Order Not Received by Lab

  • Check order status - may be pending transmission
  • Verify lab's electronic ordering is functioning
  • Print requisition and fax to lab as backup
  • Call lab to confirm receipt

Results Not Appearing

  • Verify specimen was collected
  • Check with lab on processing status
  • Confirm lab has correct patient identifiers
  • Check interface logs (Admin only)

Can't Find Specific Test

  • Search by CPT code instead of test name
  • Check if test is in selected lab's menu
  • Use "Other/Custom Test" option and enter details manually
  • Contact lab to add test to catalog
